Submission date: 2025-12-20T19:19:09
Course: COMS W4111 Introduction to Databases
Professor: Kenneth Ross
Rating: 4
Review text:
If you're hoping to learn SQL and how to use databases, this class does that... but only for the first few weeks. After that, it is all database management systems, but after the class you do have a pretty good idea of how databases are implemented which is useless for most people at the face level, but very useful for actually using databases.
Still, the material is boring. Ross lectures as if he understands this fact and has surrendered to his powerlessness in changing it. Every lecture consists of a new topic which he explains well and a few examples. The review lectures are pretty useful.
Basically, the structure, workload, and instruction of the class are all pretty good thanks to Ross, but you will be falling asleep anyway which is mostly the material's fault.
Workload notes:
Workload is not high. The only work outside of the midterm and final are 2 HW assignments, which are SQL practice, and the semester-long project has you building a full-stack application (Python/Flask) which is genuinely useful experience, especially earlier on in your CS career. There is also an option for a project without coding, though I do not know much about it except that it sounds more boring. Therefore, the homeworks are pretty useful. The midterm (15% of the grade) and final (40% of the grade) do ask a few SQL questions, but are mostly theoretical questions about DB management systems. The midterm and final averages were around a 60 for the midterm and a 45 for the final which are kind of crazy considering databases is not supposed to be a terribly difficult class. There is a policy where the final replaces the midterm but that doesn't really happen since the midterm is hard, but the final is an affront to God himself. Ross curves reasonably though, so if you are able to repress the anxiety your raw scores will undoubtedly give you, you'll be alright.

Course: COMS W4111 Introduction to Databases
Professor: Luis Gravano
Rating: 4
Review text:
Professor Gravano is overall a great Professor. In this class, you will learn mostly about SQL databases. You will learn how to design a proper database schema which enforces integrity constraints, how to implement this schema using PostgreSQL, and how to use and interact with a database server. These are all very useful for people who want to go into software engineering
There are also some more niche topics covered (e.g. how database locks work to ensure atomic transactions) which are not as practical, but still very helpful.
Workload notes:
There are two exams (one midterm and one final) which are reasonable and fairly similar to the practice exams made available. There is also one big project throughout most of the semester, which has several parts. The project consists on designing a database for an application, implementing it in SQL, populating it with data, and ultimately building a simple web app that uses it. This project was extremely helpful to understand how to use SQL databases in practice. The grading for the project is very forgiving. Just turn in everything on time and you will be fine
